Solidity Basics

Let's start with a Hello World contract, and go over some of its key features:

After the first few lines of the file (which specify the Solidity compiler version and provide a multi-line comment), this codes shouldn't look unfamiliar to developers with experience with JavaScript. Contracts are essentially classes, which contain their own internal state. As we'll see, contracts have public and private functions, contain instance variables, and can inherit from other contracts.

The other thing that jumps out is that everything in Solidity is statically typed, with the type coming before the variable definition. We have strings, bytes, booleans, and different sized ints and uints, all with the basic operators available in JavaScript. Additionally, Solidity has a special type to denote Ethereum addresses (ex. 0xcA5228D1fe52D22db85E02CA305cddD9E573D752).

Solidity also has a native mapping type, which stores key/value pairs of pretty much any type. Likewise, it has arrays and structs.

Read more in Solidity types in the docs.

In this example, we create a FastCashMoneyPlus contract and set several public instance variables. totalSupply and fastCashBank (which represent the total outstanding balance of FastCash and the amount of unallocated FastCash, respectively) are left empty for now, while decimals and name are set to specific values. On line 13, we create a mapping of addresses to uint, representing the balance of each FastCash account. Finally on line 15, we define a constructor function (which always shares the name of the contract), and totalSupply and fastCashBank to 1 million FastCash.

You might notice that something funny is going on regarding decimal places: Solidity doesn't have floating point numbers! If this is the case, how can we transact fractions of FastCash?? While it may seem a bit rounsabout, the standard way to handle this is to specify a special decimals instance variable that, as you might guess, indicates the number of "implicit" decimal places in your coin. This means that the smallest denomination of FastCash (which is called the MoneyBuck) is equal to 0.000000000000000001 FastCash, and that totalSupply equals 1000000000000000000000000 MoneyBucks!
